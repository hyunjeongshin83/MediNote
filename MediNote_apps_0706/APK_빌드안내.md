# APK 자동 빌드 사용법 (GitHub Actions)

이 저장소에는 **코드를 올리면 APK가 자동으로 만들어지는** 설정이 들어 있습니다.
컴퓨터에 개발 도구를 깔지 않아도, GitHub가 클라우드에서 대신 빌드해 줍니다.

## 어떻게 작동하나요

- `medinote_apps` 폴더를 저장소에 올리면(push), 자동으로 빌드가 시작됩니다.
- 설정 파일 위치: `.github/workflows/android-build.yml`

## APK 내려받는 방법

1. GitHub 저장소 화면 위쪽의 **Actions** 탭을 누릅니다.
2. 최근 실행된 **Build Android APK** 항목을 누릅니다.
3. 초록색 체크(성공)가 뜨면, 아래 **Artifacts** 에서 **MediNote-debug-apk** 를 내려받습니다.
4. 압축을 풀면 `app-debug.apk` 파일이 나옵니다. 이 파일을 안드로이드 폰에 옮겨 설치할 수 있습니다.
   - 폰에서 "출처를 알 수 없는 앱 설치 허용"을 켜야 설치됩니다(테스트용).

## 수동으로 빌드하고 싶을 때

- Actions 탭 → **Build Android APK** → 오른쪽 **Run workflow** 버튼을 눌러 언제든 다시 빌드할 수 있습니다.

## 안내

- 이렇게 만든 APK는 **테스트용(디버그) 빌드**입니다. 지인·연구자에게 나눠 주고 폰에 설치해 볼 수 있습니다.
- **구글 플레이 스토어 등록**에는 별도의 '서명된 릴리스 빌드'가 필요하며, 이는 본인의 개발자 키로 직접 만들어야 합니다. 준비되시면 그 방법도 안내드리겠습니다.
- 빌드가 처음 실행될 때는 몇 분 걸릴 수 있습니다.
