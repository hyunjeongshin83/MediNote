# MediNote (메디노트)

예방접종·복약 안내 앱과 관련 자료를 담은 저장소입니다.
복약관리와 생애주기 예방접종(어린이·성인·노인·보건의료인·해외여행)을
하나의 노트에 담은 고령친화 헬스케어 프로젝트입니다.

---

## 저장소 구성

이 저장소에는 두 개의 주요 폴더가 있습니다.

### 1. `MediNote_app_0706` — 시제품 앱과 안내 문서

초기 계획에 따라 만든 **시제품(웹앱)과 소개·안내 문서**를 모아 둔 폴더입니다.

- `MediNote_app_0706.html` — 예방접종·복약 안내 앱 시제품 (브라우저에서 열림)
- `MediNote_소개자료.pdf` (또는 `MediNote_브로셔.pdf`) — 앱 소개 자료
- `GitHub_가입안내.pdf` — 연구자용 GitHub 가입·의견 등록 안내
- `AI코드수정_안내.md` — 비개발자가 Copilot·ChatGPT·Claude로 코드를 고치는 방법

> 이 폴더는 "보여주고 의견받는" 용도입니다. 연구자·교수님이 시제품을 열어 보고,
> GitHub Issues로 의견을 남기실 수 있도록 안내 문서를 함께 담았습니다.

### 2. `MediNote_apps_0706` — 안드로이드·iOS 앱 코드

시제품을 **안드로이드·iOS 앱**으로 만드는 코드입니다.
웹앱(HTML)을 앱으로 감싸, 하나의 코드로 안드로이드·iOS·웹페이지를 모두 사용합니다.

- `android/` — 안드로이드 앱 (Kotlin, WebView) + 건강 데이터 로컬 연동
- `ios/MediNote/` — iOS 앱 (Swift, WKWebView) + 건강 데이터 로컬 연동 + iOS 빌드 안내
- `shared/index.html` — 웹앱 (웹페이지로도 사용)
- `APK_빌드안내.md` — GitHub에서 APK 자동으로 만드는 방법
- 자세한 내용은 `MediNote_apps_0706/README.md` 참고

### 자동 빌드 설정

- `.github/workflows/android-build.yml` — 저장소 **최상위**에 있어야 하며,
  코드를 올리면 GitHub가 APK를 자동으로 만들어 줍니다. (Actions 탭에서 내려받기)

## 폴더 이름 바꾸는 방법 (GitHub 웹)

GitHub 웹에서는 폴더를 직접 "이름 변경"하는 버튼이 없어서,
**폴더 안 파일의 경로를 바꾸는 방식**으로 이름을 바꿉니다.

1. 이름을 바꿀 폴더 안의 **파일 하나**를 엽니다.
2. 오른쪽 위 **연필(Edit) 아이콘**을 누릅니다.
3. 파일 이름 입력칸의 맨 앞(경로 부분)에서, **폴더 이름을 새 이름으로 고칩니다.**
   - 예: `MediNote_app_0706/파일.html` → `메디노트_앱/파일.html`
   - 슬래시(/)를 입력하면 폴더 경로가 됩니다.
4. 아래 **Commit changes**를 누릅니다.
5. 같은 폴더의 나머지 파일도 같은 방식으로 경로를 바꾸면,
   기존 폴더는 비게 되고 새 이름의 폴더로 옮겨집니다.

> 파일이 많으면 번거로우므로, **컴퓨터에서 폴더 이름을 바꾼 뒤 통째로 다시 업로드**하는 방법이
> 더 쉬울 수 있습니다. (Add file → Upload files 로 새 이름의 폴더를 올림)

> 참고: 폴더 이름을 바꾸면, `.github/workflows/android-build.yml` 안의
> `working-directory` 경로도 새 폴더 이름에 맞게 바꿔야 APK 빌드가 계속 작동합니다.

## 안내

- 본 앱은 개념검증 시제품이며, 화면의 접종·복약 정보는 예시 데이터입니다.
- 접종 정보는 판정이 아니라 안내와 상담 연결까지만 제공합니다.
- 건강 센서 데이터는 민감정보로, 실제 연결 전에는 예시 데이터로 흐름만 시연하며
  별도 동의와 법적 검토를 갖춘 뒤 진행합니다.
